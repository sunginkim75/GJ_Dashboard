# 구글 API 및 스프레드시트 연동 가이드

본 가이드는 경조사 관리 시스템이 구글 스프레드시트 데이터를 읽고 쓰고, 구글 로그인을 처리할 수 있도록 구글 클라우드 콘솔을 설정하는 방법을 안내합니다.

---

## 1. 구글 서비스 계정(Service Account) 생성 (스프레드시트 연동용)

개발하는 웹 앱이 백엔드에서 스프레드시트에 접근하기 위해서는 서비스 계정이 필요합니다.

1. **[구글 클라우드 콘솔](https://console.cloud.google.com/)**에 접속하여 로그인합니다.
2. 프로젝트가 없다면 상단에서 **[프로젝트 만들기]**를 클릭해 새 프로젝트를 생성합니다.
3. 왼쪽 메뉴에서 **[API 및 서비스] > [라이브러리]**로 이동합니다.
4. 검색창에 다음 두 API를 각각 검색하여 **[사용(Enable)]** 설정합니다:
   - **Google Sheets API**
   - **Google Drive API**
5. 왼쪽 메뉴에서 **[IAM 및 행정] > [서비스 계정]**으로 이동합니다.
6. 상단의 **[+ 서비스 계정 만들기]**를 클릭합니다.
   - **서비스 계정 이름**: `gj-dashboard-handler` (원하는 이름 입력)
   - **만들고 계속하기** 및 **완료**를 클릭해 생성을 마칩니다.
7. 생성된 서비스 계정의 이메일 주소(예: `gj-dashboard-handler@...iam.gserviceaccount.com`)를 복사해 둡니다.
8. 해당 서비스 계정을 클릭한 뒤, 상단 탭에서 **[키(Keys)]**로 이동합니다.
9. **[키 추가] > [새 키 만들기]**를 선택하고 형식을 **JSON**으로 하여 만듭니다.
10. 다운로드된 JSON 파일의 이름을 `credentials.json`으로 변경하여 프로젝트의 `config/credentials.json` 경로에 배치해야 합니다.

---

## 2. 구글 스프레드시트 공유 설정

경조사 관리 스프레드시트에 서비스 계정이 접근할 수 있도록 권한을 부여해야 합니다.

1. 연동할 [구글 스프레드시트](https://docs.google.com/spreadsheets/d/1wwB0_rMS3uQrhRykRp4kdrzV0bWHDqwjniuSuBdztzk/edit)를 엽니다.
2. 오른쪽 상단의 **[공유]** 버튼을 클릭합니다.
3. 위 **1단계의 7번**에서 복사해 둔 서비스 계정 이메일 주소(`gj-dashboard-handler@...iam.gserviceaccount.com`)를 입력합니다.
4. 권한을 **편집자(Editor)**로 지정하고 **[전송]**을 누릅니다.

---

## 3. 구글 OAuth 2.0 클라이언트 ID 생성 (구글 로그인 인증용)

사용자 본인 인증을 위해 구글 OAuth ID가 필요합니다.

1. 구글 클라우드 콘솔의 **[API 및 서비스] > [OAuth 동의 화면]**으로 이동합니다.
2. User Type을 **[외부(External)]**로 선택하고 **[만들기]**를 누릅니다.
   - 앱 이름, 사용자 지원 이메일, 개발자 연락처 등을 적고 **[저장하고 계속하기]**를 누릅니다. (나머지 범위 및 테스트 사용자는 기본값으로 둡니다.)
3. **[API 및 서비스] > [사용자 인증 정보]** 메뉴로 이동합니다.
4. 상단의 **[+ 사용자 인증 정보 만들기] > [OAuth 클라이언트 ID]**를 선택합니다.
5. 애플리케이션 유형을 **[웹 애플리케이션]**으로 선택합니다.
   - **이름**: `GJ Dashboard Login`
   - **승인된 JavaScript 원본**:
     - `http://localhost:8000` (로컬 테스트용)
     - `http://127.0.0.1:8000`
   - **승인된 리디렉션 URI**:
     - `http://localhost:8000/api/auth/callback`
     - `http://127.0.0.1:8000/api/auth/callback`
6. **[만들기]**를 누르면 **클라이언트 ID**와 **클라이언트 보안 비밀번호**가 생성됩니다.
7. 생성된 클라이언트 ID를 복사해 두고, 추후 환경설정(`config/settings.py` 등)에 등록합니다.
